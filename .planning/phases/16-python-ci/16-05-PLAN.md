  description: 'Run firmware tests in the LECERF Cortex-M emulator inside a Docker container'

      description: 'Path to directory containing pytest test_*.py files'
      required: true

      description: 'Per-test instruction budget'
      required: false
      default: '5000'
  outputs:
    junit-xml:
      description: 'Path to JUnit XML report inside the container'
      value: '/tmp/junit.xml'
  runs:
    using: 'docker'
    image: 'docker://ghcr.io/korsarofficial/lecerf:latest'
    args:
      - ${{ inputs.firmware-dir }}
      - ${{ inputs.tests-dir }}

Create .github/workflows/release.yml (~70 lines):

  name: release
  on:
    push:
      tags: ['v*']

  jobs:
    build-wheel-linux:
      runs-on: ubuntu-22.04
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-python@v5
          with: { python-version: '3.12' }
        - name: install build deps
          run: |
              sudo apt-get update
              sudo apt-get install -y cmake build-essential
              pip install build scikit-build-core
        - name: build wheel
          run: |
              cd python
              python -m build --wheel
        - uses: actions/upload-artifact@v4
          with: { name: wheels, path: python/dist/*.whl }

    build-and-push-docker:
      runs-on: ubuntu-22.04
      permissions:
        contents: read
        packages: write
      steps:
        - uses: actions/checkout@v4
        - uses: docker/login-action@v3
          with:
              registry: ghcr.io
              username: ${{ github.actor }}
              password: ${{ secrets.GITHUB_TOKEN }}
        - uses: docker/build-push-action@v5
          with:
              context: .
              file: docker/Dockerfile
              push: true
              tags: |
                  ghcr.io/${{ github.repository_owner }}/lecerf:latest
                  ghcr.io/${{ github.repository_owner }}/lecerf:${{ github.ref_name }}

    release:
      needs: [build-wheel-linux, build-and-push-docker]
      runs-on: ubuntu-22.04
      permissions:
        contents: write
      steps:
        - uses: actions/download-artifact@v4
          with: { name: wheels, path: dist/ }
        - uses: softprops/action-gh-release@v2
          with:
              files: dist/*.whl
              generate_release_notes: true

Notes:
- Single-platform wheel for MVP (Linux x86_64). cibuildwheel matrix is post-MVP
  (deferred to phase 17).
- ghcr.io is used because GITHUB_TOKEN auth is automatic; no Docker Hub account
  required.
- Docker build pulls the same Dockerfile from 16-04 -- no duplication.

  # lecerf-ci-example

  Sample firmware test suite for LECERF — the Cortex-M emulator built for CI.

  ## What this shows

  Three firmware binaries are exercised by three pytest tests. CI runs the
  full suite in &lt; 30 seconds on a stock GitHub-hosted runner.

  ## 5-minute onboarding

  1. Clone this directory and push it to a fresh GitHub repo.
  2. The included `.github/workflows/firmware-test.yml` calls the LECERF
     Docker-container Action; on first push, the runner pulls the image
     (~45 MB) and runs `pytest tests/`.
  3. Watch the green check appear under Actions tab.

  ## Layout

  ```
  firmware/
    test1.bin                    # fibonacci -> R0 = 55
    test3.bin                    # UDIV / MUL -> UART output
    test10_stm32_blink.bin       # GPIO toggle, R0 = 5
  tests/
    test_fib.py                  # asserts R0 == 55 after fib firmware
    test_uart.py                 # asserts UART bytes captured
    test_blink.py                # asserts blink loop counter == 5
  .github/workflows/firmware-test.yml
  requirements.txt
  ```

  ## Local run (no Docker)

  ```bash
  pip install -r requirements.txt
  pytest tests/
  ```

  ## Local run (with Docker, identical to CI)

  ```bash
  docker run --rm \
      -v "$(pwd)/firmware:/fw:ro" \
      -v "$(pwd)/tests:/tests:ro" \
      ghcr.io/korsarofficial/lecerf:latest /fw /tests
  ```

  ## Adding your own firmware test

  1. Drop `myfw.bin` into `firmware/`.
  2. Create `tests/test_myfw.py`:
     ```python
     from .conftest import fwpath

     def test_myfw_halts(board):
         r = board.flash(fwpath("myfw")).run(timeout_ms=1000)
         assert r.exit_cause == "halt"
         # add your assertions on r.r[N], board.uart.output(), board.gpio[..][..]
     ```
  3. Push. CI runs it.

  ## License: MIT
