<objective>
Add pseudo-chaining to the JIT dispatch path. Currently every block return takes a full
C-frame round-trip through run_steps_full_g (decode-cache + IRQ check + jit_run dispatch +
maybe restart). With compiled native blocks for ~80% of FreeRTOS hot code (after 14-02 LDR/
STR + 14-03 flags + 14-04 B.cond), these C-frame returns are the dominant overhead.

Pseudo-chaining (research E.18 simpler variant): a tight `while` loop in jit_run_chained
that re-dispatches through compiled blocks within the SAME C-frame. Per research H.28: the
chained loop must respect max_steps so run_until_cycle (TT-06 rewind path) keeps cycle-stamp
precision (overshoot bounded by JIT_MAX_BLOCK_LEN-1 = 31 cycles). IRQ injection stays at the
outer run_steps_full_g level (research H.30 pitfall 4) — chained loop yields control on miss.

Add JIT-05: TB cache eviction via generation reset. When compile_block fails because
n_blocks == JIT_MAX_BLOCKS, call jit_flush and retry once. Simpler than QEMU's per-slot LRU;
adequate for FreeRTOS hot-loop steady state of ~200 unique TBs (cap is 1024).

Purpose: JIT-01 (direct chaining; pseudo-chain in spirit) + JIT-05 (cache eviction) closed.
Combined with 14-02..14-04 native paths, this is the final ~3-5x IPS multiplier needed for
100M+ IPS target. 14-06 bench will measure the cumulative effect.

Output: jit_run_chained added; run_steps_full_g uses it; eviction path implemented; test
exercises max_steps budget enforcement and eviction wrap-around.
</objective>

Step B — implement in src/core/jit.c. Place after jit_run (around line 99):

```c
bool jit_run_chained(jit_t* j, cpu_t* c, bus_t* b, exec_fn execute,
                     u64 max_steps, u64* out_steps) {
    u64 total = 0u;
    while (!c->halted && total < max_steps) {
        u64 budget = max_steps - total;
        u64 steps = 0u;
        if (!jit_run(j, c, b, execute, &steps)) break;
        if (steps == 0u) break;                 /* defensive: no forward progress */
        total += steps;
        (void)budget;                            /* current jit_run runs whole block */
    }
    *out_steps = total;
    return total > 0u;
}
```

Note: the inner jit_run runs an entire block (up to JIT_MAX_BLOCK_LEN=32 ARM insns) atomically
once entered. We can NOT mid-block stop. So overshoot is bounded by 31 cycles. This is
acceptable for run_until_cycle: TT-06 already tolerates +1 ARM cycle in step_back, and the
event-injection precision in tt_replay is preserved because run_until_cycle's outer loop
re-checks log[*pos].cycle after each chained call (research H.28).

To minimize overshoot when budget is tight, we could check `total + JIT_MAX_BLOCK_LEN > max_steps`
and break early — but that would prematurely abandon the chained path even when the next block
might be shorter than 32 insns. Simpler heuristic: when budget < JIT_MAX_BLOCK_LEN, exit chain
and let the outer loop dispatch single insns:

```c
bool jit_run_chained(jit_t* j, cpu_t* c, bus_t* b, exec_fn execute,
                     u64 max_steps, u64* out_steps) {
    u64 total = 0u;
    while (!c->halted && total < max_steps) {
        u64 remaining = max_steps - total;
        if (remaining < (u64)JIT_MAX_BLOCK_LEN) break;
        u64 steps = 0u;
        if (!jit_run(j, c, b, execute, &steps)) break;
        if (steps == 0u) break;
        total += steps;
    }
    *out_steps = total;
    return total > 0u;
}
```

This caps overshoot at exactly JIT_MAX_BLOCK_LEN-1 = 31 cycles when remaining hits the cliff.
Outer loop continues with interpreter for the last <32 insns until target reached.

Step C — eviction in compile_block. Current compile_block (jit.c:37-58) returns NULL when
n_blocks >= JIT_MAX_BLOCKS. After NULL return, jit_run treats as "compile failed -> fallback"
which means the block won't be cached. Permanent thrash if cache is full.

Patch compile_block to do generation reset on overflow:

```c
static jit_block_t* compile_block(jit_t* j, bus_t* b, u32 pc) {
    if (j->n_blocks >= JIT_MAX_BLOCKS) {
        /* Cache full: generation reset. Wipes all blocks, counters, codegen buffer.
           Lookup tables back to -1. Fresh block compilation begins from slot 0. */
        jit_flush(j);
    }
    /* (rest of body unchanged) */
    jit_block_t* bk = &j->blocks[j->n_blocks];
    bk->pc_start = pc;
    bk->n_ins = 0;
    bk->native = NULL;
    u32 cur = pc;
    while (bk->n_ins < JIT_MAX_BLOCK_LEN) {
        insn_t ins;
        decode(b, cur, &ins);
        ins.pc = cur;
        bk->ins[bk->n_ins++] = ins;
        cur += ins.size;
        if (is_terminator(ins.op)) break;
    }
    bk->pc_end = cur;
    bk->native = codegen_emit(&j->cg, bk->ins, bk->n_ins);
    install(j, pc, (int)j->n_blocks);
    j->n_blocks++;
    return bk;
}
```

Note: jit_flush resets counters[] too, so the re-promotion path (counter must reach
JIT_HOT_THRESHOLD before compile) RESTARTS for every PC after eviction. That's fine — hot
PCs will rapidly re-promote on subsequent passes.

Also: after jit_flush, j->cg.used = 0, so codegen_emit has the full 2MB buffer to fill again.

Step D — codegen.c capacity check. After 14-04, codegen_emit checks `cg->used + n*96 + 64
> cg->capacity` and returns NULL if it would overflow the 2MB codegen buffer (CG_BUFFER_SIZE).
This is independent of n_blocks. If we hit codegen-buffer overflow before n_blocks fills,
the block won't be native (just interpreted from the cached insn[]). Acceptable for now;
research G.20 confirms 1024 slots × ~512B = 512KB, well under 2MB. So codegen overflow only
hits if 100% of blocks compile to large thunks (e.g., heavy LDR/STR), and even then we'd hit
n_blocks=1024 first.

u64 run_steps_full_g(cpu_t* c, bus_t* bus, u64 max_steps,
                     systick_t* st, scb_t* scb, jit_t* g) {
    dcache_invalidate();
    u64 i = 0;
    for (; i < max_steps && !c->halted; ++i) {
        u64 jit_steps = 0;
        u64 budget = max_steps - i;            /* remaining budget for chain */
        if (g && jit_run_chained(g, c, bus, execute, budget, &jit_steps) && jit_steps > 0) {
            if (st) systick_tick(st, (u32)jit_steps);
            if (g_dwt_for_run) for (u64 k = 0; k < jit_steps; ++k) dwt_tick(g_dwt_for_run);
            i += jit_steps - 1;
            goto check_irqs;
        }
        insn_t ins;
        cache_decode(bus, c->r[REG_PC], &ins);
        if (!execute(c, bus, &ins)) break;

        if (st) systick_tick(st, 1);
        if (g_dwt_for_run) dwt_tick(g_dwt_for_run);

    check_irqs:
        if (c->mode == MODE_THREAD && !(c->primask & 1u)) {
            if (st && st->irq_pending) {
                st->irq_pending = false;
                exc_enter(c, bus, EXC_SYSTICK);
                continue;
            }
            if (scb && scb->pendsv_pending) {
                scb->pendsv_pending = false;
                exc_enter(c, bus, EXC_PENDSV);
                continue;
            }
            if (g_nvic_for_run) {
                int irq = nvic_pick(g_nvic_for_run);
                if (irq >= 0) {
                    nvic_clear_pending(g_nvic_for_run, (u32)irq);
                    nvic_set_active(g_nvic_for_run, (u32)irq);
                    exc_enter(c, bus, (u8)(EXC_IRQ0 + irq));
                    continue;
                }
            }
        }
    }
    return i;
}

  <name>Task 3: max_steps budget + eviction wrap test</name>
  <files>tests/test_jit_chain.c, tests/CMakeLists.txt</files>
  <action>
Goal: prove three properties:
1. Chained dispatch runs >1 block within a single jit_run_chained call (verify by running
   a back-to-back pair of native-eligible blocks and checking out_steps > MAX_BLOCK_LEN).
2. max_steps budget honored: passing max_steps=1 returns within MAX_BLOCK_LEN insns.
3. Eviction wrap: after JIT_MAX_BLOCKS unique PCs are compiled, the (1025th) PC's compile
   triggers jit_flush + recompile, and the resulting block is native (n_blocks reset to 1).

```c
#include "test_harness.h"
#include "core/cpu.h"
#include "core/bus.h"
#include "core/codegen.h"
#include "core/decoder.h"
#include "core/jit.h"
#include <string.h>

extern bool execute(cpu_t* c, bus_t* bus, const insn_t* i);

static jit_t s_jit;

/* Build a tiny self-contained ARM "program" in flash:
     0x0000:  MOV r0, #0      (T1, 2B)         pc=0x0
     0x0002:  ADD r0, #1      (T1 ADD_IMM8 rd=0 imm=1, 2B)   pc=0x2
     0x0004:  B   #-6         (T1 B_UNCOND, target=0x0, imm=-8) pc=0x4
   This is a hot infinite loop. Native-eligible from 14-04 (B_UNCOND) + native ADD/MOV. */
static void build_loop_blob(bus_t* bus) {
    /* Hand-encode in flash bytes: */
    region_t* fl = bus_find_flat(bus, 0x00000000u);
    if (!fl) return;
    /* MOV r0,#0  -> 16-bit Thumb imm: 0010 0 ddd iiiiiiii  (MOVS) -> rd=0,imm=0  -> 2000 (LE) */
    fl->buf[0] = 0x00; fl->buf[1] = 0x20;
    /* ADD r0,#1  -> 0011 0 ddd iiiiiiii  rd=0 imm=1 -> 0x3001 */
    fl->buf[2] = 0x01; fl->buf[3] = 0x30;
    /* B label  -> 11100 sssssssssss  imm11 = -4 (loop back) -> imm = -4*2 = -8 in bytes
                  In Thumb-1 B_UNCOND: target = pc(B)+4+imm. pc=4, want target=0, so imm = -8.
                  imm11 (signed 11-bit) = -4 -> 0x7FC. Encoding: 11100 11111111100 -> 0xE7FC */
    fl->buf[4] = 0xFC; fl->buf[5] = 0xE7;
}

int main(void) {
    bus_t bus; bus_init(&bus);
    bus_add_flat(&bus, "flash", 0x00000000u, 65536u, false);
    bus_add_flat(&bus, "sram",  0x20000000u, 4096u, true);
    /* Override "writable" for flash this once so we can patch the loop blob: */
    region_t* fl = bus_find_flat(&bus, 0x00000000u);
    if (fl) fl->writable = true;
    build_loop_blob(&bus);
    if (fl) fl->writable = false;

    cpu_t cpu = {0};
    cpu_reset(&cpu, CORE_M4);
    cpu.r[REG_PC] = 0u;
    cpu.r[REG_SP] = 0x20001000u;

    memset(&s_jit, 0, sizeof s_jit);
    jit_init(&s_jit);

    /* Force the block to be hot. JIT_HOT_THRESHOLD = 16. Probe with single-block jit_run
       16 times so the block compiles. */
    for (int probe = 0; probe < 32; ++probe) {
        u64 steps = 0;
        (void)jit_run(&s_jit, &cpu, &bus, execute, &steps);
        if (cpu.halted) break;
    }

    /* Test 1: chained call should run thousands of insns in one go */
    cpu.r[REG_PC] = 0u; cpu.cycles = 0u; cpu.halted = false;
    cpu.r[0] = 0u;
    u64 steps = 0;
    bool ok = jit_run_chained(&s_jit, &cpu, &bus, execute, 10000ull, &steps);
    TEST(ok, "chain run failed");
    TEST(steps >= (u64)JIT_MAX_BLOCK_LEN * 2u, "chain ran fewer than 2 blocks");
    TEST(cpu.r[0] >= 1u, "ADD did not execute (r0 should increment)");

    /* Test 2: max_steps budget */
    cpu.r[REG_PC] = 0u; cpu.r[0] = 0u;
    u64 small = 0;
    (void)jit_run_chained(&s_jit, &cpu, &bus, execute, (u64)JIT_MAX_BLOCK_LEN - 1u, &small);
    TEST(small == 0u || small <= (u64)JIT_MAX_BLOCK_LEN, "tiny budget overshoots");

    /* Test 3: eviction wrap. Force n_blocks to JIT_MAX_BLOCKS-1 and compile one more block,
       then verify the next compile triggers jit_flush. */
    s_jit.n_blocks = (u32)JIT_MAX_BLOCKS;
    /* Try to compile_block via jit_run by probing a fresh PC: warm a new PC enough to compile */
    cpu.r[REG_PC] = 0x100u;        /* unmapped, but compile path doesn't read PC for decode failure */
    /* Actually we need PC to point at valid Thumb in flash. Re-use loop entry with a different cache slot: */
    cpu.r[REG_PC] = 0u;
    /* Probe enough to retrigger compile */
    for (int probe = 0; probe < 32; ++probe) {
        u64 dummy = 0;
        (void)jit_run(&s_jit, &cpu, &bus, execute, &dummy);
        if (cpu.halted) break;
    }
    /* After eviction: n_blocks should be ~1 (just the freshly compiled block) */
    TEST(s_jit.n_blocks <= 2u, "eviction did not reset n_blocks");

    PASS();
}
```
