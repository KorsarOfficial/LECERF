Add native NZCV flag updates to all ALU ops that the interpreter sets flags for. Until now,
native ADD/SUB/AND/etc. produce the wrong APSR — research C.10 confirms ADD_REG silently
DOES NOT update flags in the JIT path, masked only because CMP/B.cond fall back to
interpreter today (which corrects flags before the conditional). Once 14-04 lands native
B.cond, the bug becomes user-visible. Fix now.

Add native CMP / CMN / TST family which currently force interpreter fallback for any block
containing them.

Bit-mapping (research C.11):
- ARM N (apsr bit 31)  <-  x86 SF
- ARM Z (apsr bit 30)  <-  x86 ZF
- ARM C (apsr bit 29)  <-  x86 CF for ADD; NOT(CF) for SUB
- ARM V (apsr bit 28)  <-  x86 OF
- AND/ORR/EOR (NZ-only): leave C and V unchanged; just update bits 30,31

Sequence: x86 instr (add/sub/and) -> lahf (loads SF:ZF:0:AF:0:PF:1:CF into AH) -> seto cl
(captures OF into cl). Then assemble target NZCV bits via shifts/ors and merge into apsr
with one read-modify-write at [r15+APSR_OFF].

The flag emit must run AFTER the x86 arithmetic instruction (add/sub/and/etc.) sets EFLAGS.
The result is in eax. Do not touch eax until flags are sampled. Sequence (research C.12 +
section 9 pitfall 7 corrections):

```asm
; ASSUMPTION: x86 ALU op (add eax,ecx | sub eax,ecx | and eax,ecx) just executed.
; eax = result; EFLAGS holds SF/ZF/CF/OF for the just-completed op.

lahf                          ; 9F                AH = SF:ZF:0:AF:0:PF:1:CF (bits 7,6,4,2,0)
seto cl                       ; 0F 90 C1          cl = OF (1 if signed overflow)
; build NZCV mask in edx (start clean):
movzx edx, ah                 ; 0F B6 D4          edx = AH (bits 0..7 = SF:ZF:0:AF:0:PF:1:CF)
;   edx contains: bit7=SF, bit6=ZF, bit0=CF
; assemble:
mov  esi, edx
shr  esi, 7                   ; esi[0] = SF
shl  esi, 31                  ; esi = N << 31
mov  edi, edx
shr  edi, 6
and  edi, 1                   ; edi[0] = ZF
shl  edi, 30                  ; edi = Z << 30
or   esi, edi
mov  edi, edx
and  edi, 1                   ; edi[0] = CF
; NOTE: for ADD: ARM C = CF. For SUB: ARM C = NOT CF. Caller decides.
; ... (caller XORs edi with 1 if SUB before next shift) ...
shl  edi, 29                  ; edi = C << 29
or   esi, edi
movzx edi, cl                 ; edi = OF (0 or 1)
shl  edi, 28                  ; edi = V << 28
or   esi, edi
; merge into APSR (preserve Q bit at 27 and bits 0..26):
mov  edi, [r15 + CG_APSR_OFF]
and  edi, 0x0FFFFFFF          ; clear N,Z,C,V (top 4 bits)
or   edi, esi
mov  [r15 + CG_APSR_OFF], edi
```

We MUST avoid clobbering rdi (it is non-volatile in WIN64 but we are using `edi` register
locally — that IS rdi's low 32 bits, and would corrupt the caller's saved value). Wait —
we are inside a thunk, the caller of this thunk is jit_run; the thunk's prologue does NOT
save rdi (it saves r15/r14/rbx/rsi). After the thunk's prologue, rdi still holds whatever
jit_run left in it (which is cpu_t* — but that's irrelevant inside the thunk). We can use
edi/rdi freely; we're not preserving caller state because rdi is a volatile (caller-saved)
register in WIN64.

CHECK: WIN64 volatile/scratch: rax, rcx, rdx, r8, r9, r10, r11, AND THE SSE registers. rdi
is NON-VOLATILE in WIN64 (per Microsoft x64 ABI). So if jit_run is the C caller, jit_run's
own compiled code expects rdi to survive the thunk call. When jit_run was compiled by
mingw-gcc with WIN64 ABI, it WILL spill rdi if it needs the value across the call.

Conclusion: rdi IS callee-saved in WIN64. If we modify rdi inside the thunk WITHOUT saving
it on the stack, we corrupt the caller. We MUST either push rdi at prologue or use only
volatile registers for our scratch.

Available volatile regs we haven't used: rax (used for result), rcx (used as helper arg /
scratch), rdx (used as addr in LDR/STR), r8, r9, r10, r11. r10 and r11 are FREE volatile
scratch in WIN64. Use r10 and r11 instead of esi/edi.

Wait — but emit_prologue from 14-01 already pushes rsi (`56`) and rbx (`53`). We DO save
those, so esi/ebx are safe to clobber inside the thunk. ebx is our failure flag (set by
14-02's emit_clear_fail / or bl,1). esi is FREE inside the thunk after prologue.

So: use esi for one scratch (already saved by prologue), and r10d for another scratch
(volatile, no save needed).

Re-write the flag sequence using esi (saved by prologue) and r10d (volatile):

```asm
lahf                                      ; 9F
seto cl                                   ; 0F 90 C1
movzx edx, ah                             ; 0F B6 D4   edx = AH

; N = SF >> 7 then << 31:
mov  esi, edx                             ; 89 D6
shr  esi, 7                               ; C1 EE 07
shl  esi, 31                              ; C1 E6 1F     esi = N << 31

; Z = (AH>>6) & 1 << 30
mov  r10d, edx                            ; 41 89 D2  (mov r10d,edx)  3B
shr  r10d, 6                              ; 41 C1 EA 06              4B
and  r10d, 1                              ; 41 83 E2 01              4B
shl  r10d, 30                             ; 41 C1 E2 1E              4B
or   esi, r10d                            ; 44 09 D6                 3B

; C = AH & 1 (with optional XOR for SUB):
mov  r10d, edx                            ; 41 89 D2  3B
and  r10d, 1                              ; 41 83 E2 01  4B
;  -- caller may emit `xor r10d, 1` here (3B: 41 83 F2 01) for SUB --
shl  r10d, 29                             ; 41 C1 E2 1D  4B
or   esi, r10d                            ; 44 09 D6  3B

; V = OF << 28
movzx r10d, cl                            ; 44 0F B6 D1  4B
shl  r10d, 28                             ; 41 C1 E2 1C  4B
or   esi, r10d                            ; 44 09 D6  3B

; merge into apsr:
mov  r10d, [r15 + CG_APSR_OFF]            ; 45 8B 97 disp32  7B
and  r10d, 0x0FFFFFFF                     ; 41 81 E2 FF FF FF 0F  7B
or   r10d, esi                            ; 44 09 D2  ... wait, that writes esi into r10d?
;       or r/m32, r32:  09 /r;  with REX.R=1, dst=r10d, src=esi -> 41 09 F2 (or r10d, esi)
emit:                                      41 09 F2  3B
mov  [r15 + CG_APSR_OFF], r10d            ; 45 89 97 disp32   7B
```

Hmm, the OR r10d, esi opcode: `09 /r` is `or r/m, reg`, so `or r/m=r10d, reg=esi` =
`41 09 F2` (REX.B=1 for r/m=r10, ModRM 0xF2 = mod=11 reg=110=esi rm=010=r10). Wait,
checking: `09 D6` is `or esi, edx` (ModRM D6 = mod=11 reg=010=edx rm=110=esi). For `or r10d, esi`: ModRM mod=11 reg=110=esi rm=010=r10 with REX.B=1 -> 41 09 F2. Yes.

Actually simpler: do the merge with esi as the accumulator and only at the end move into r10d:
```asm
mov  r10d, [r15 + CG_APSR_OFF]
and  r10d, 0x0FFFFFFF
or   r10d, esi              ; r10d = (apsr & 0x0FFFFFFF) | esi
mov  [r15 + CG_APSR_OFF], r10d
```

```c
static void emit_lahf_seto_cl(codegen_t* cg) {
    emit_b(cg, 0x9F);                                     /* lahf */
    emit_b(cg, 0x0F); emit_b(cg, 0x90); emit_b(cg, 0xC1); /* seto cl */
}
/* movzx edx, ah  (0F B6 D4) */
static void movzx_edx_ah(codegen_t* cg) {
    emit_b(cg, 0x0F); emit_b(cg, 0xB6); emit_b(cg, 0xD4);
}
/* mov esi, edx  (89 D6) */
static void mov_esi_edx(codegen_t* cg) {
    emit_b(cg, 0x89); emit_b(cg, 0xD6);
}
/* shr esi, imm8  (C1 EE imm8) */
static void shr_esi(codegen_t* cg, u8 n) {
    emit_b(cg, 0xC1); emit_b(cg, 0xEE); emit_b(cg, n);
}
/* shl esi, imm8  (C1 E6 imm8) */
static void shl_esi(codegen_t* cg, u8 n) {
    emit_b(cg, 0xC1); emit_b(cg, 0xE6); emit_b(cg, n);
}
/* mov r10d, edx  (41 89 D2) */
static void mov_r10d_edx(codegen_t* cg) {
    emit_b(cg, 0x41); emit_b(cg, 0x89); emit_b(cg, 0xD2);
}
/* shr r10d, imm8  (41 C1 EA imm8) */
static void shr_r10d(codegen_t* cg, u8 n) {
    emit_b(cg, 0x41); emit_b(cg, 0xC1); emit_b(cg, 0xEA); emit_b(cg, n);
}
/* and r10d, imm32  (41 81 E2 imm32) */
static void and_r10d_imm(codegen_t* cg, u32 v) {
    emit_b(cg, 0x41); emit_b(cg, 0x81); emit_b(cg, 0xE2); emit_w32(cg, v);
}
/* shl r10d, imm8  (41 C1 E2 imm8) */
static void shl_r10d(codegen_t* cg, u8 n) {
    emit_b(cg, 0x41); emit_b(cg, 0xC1); emit_b(cg, 0xE2); emit_b(cg, n);
}
/* or esi, r10d  (44 09 D6) */
static void or_esi_r10d(codegen_t* cg) {
    emit_b(cg, 0x44); emit_b(cg, 0x09); emit_b(cg, 0xD6);
}
/* xor r10d, imm32  (41 81 F2 imm32) — used for ARM_C = NOT CF on SUB */
static void xor_r10d_imm(codegen_t* cg, u32 v) {
    emit_b(cg, 0x41); emit_b(cg, 0x81); emit_b(cg, 0xF2); emit_w32(cg, v);
}
/* movzx r10d, cl  (44 0F B6 D1) */
static void movzx_r10d_cl(codegen_t* cg) {
    emit_b(cg, 0x44); emit_b(cg, 0x0F); emit_b(cg, 0xB6); emit_b(cg, 0xD1);
}
/* mov r10d, [r15 + CG_APSR_OFF]  (45 8B 97 disp32) */
static void ld_r10d_apsr(codegen_t* cg) {
    emit_b(cg, 0x45); emit_b(cg, 0x8B); emit_b(cg, 0x97);
    emit_w32(cg, CG_APSR_OFF);
}
/* mov [r15 + CG_APSR_OFF], r10d  (45 89 97 disp32) */
static void st_r10d_apsr(codegen_t* cg) {
    emit_b(cg, 0x45); emit_b(cg, 0x89); emit_b(cg, 0x97);
    emit_w32(cg, CG_APSR_OFF);
}
/* or r10d, esi  (41 09 F2) */
static void or_r10d_esi(codegen_t* cg) {
    emit_b(cg, 0x41); emit_b(cg, 0x09); emit_b(cg, 0xF2);
}

/* Emit NZCV update after x86 ADD/SUB. is_sub: true if op was sub (so ARM_C = NOT CF). */
static void emit_flags_nzcv(codegen_t* cg, bool is_sub) {
    emit_lahf_seto_cl(cg);
    movzx_edx_ah(cg);
    /* N = (AH>>7) << 31  -> esi */
    mov_esi_edx(cg);
    shr_esi(cg, 7u);
    shl_esi(cg, 31u);
    /* Z = ((AH>>6) & 1) << 30  -> r10d, or into esi */
    mov_r10d_edx(cg);
    shr_r10d(cg, 6u);
    and_r10d_imm(cg, 1u);
    shl_r10d(cg, 30u);
    or_esi_r10d(cg);
    /* C = (AH & 1)  optionally XORed for SUB,  << 29  -> r10d, or into esi */
    mov_r10d_edx(cg);
    and_r10d_imm(cg, 1u);
    if (is_sub) xor_r10d_imm(cg, 1u);
    shl_r10d(cg, 29u);
    or_esi_r10d(cg);
    /* V = OF (cl) << 28  -> r10d, or into esi */
    movzx_r10d_cl(cg);
    shl_r10d(cg, 28u);
    or_esi_r10d(cg);
    /* merge into apsr */
    ld_r10d_apsr(cg);
    and_r10d_imm(cg, 0x0FFFFFFFu);
    or_r10d_esi(cg);
    st_r10d_apsr(cg);
}

/* NZ-only update (AND/ORR/EOR/MOV with set_flags). C and V unchanged. */
static void emit_flags_nz(codegen_t* cg) {
    emit_b(cg, 0x9F);                  /* lahf */
    movzx_edx_ah(cg);
    /* N = (AH>>7) << 31 */
    mov_esi_edx(cg);
    shr_esi(cg, 7u);
    shl_esi(cg, 31u);
    /* Z = ((AH>>6) & 1) << 30 */
    mov_r10d_edx(cg);
    shr_r10d(cg, 6u);
    and_r10d_imm(cg, 1u);
    shl_r10d(cg, 30u);
    or_esi_r10d(cg);
    /* preserve C and V; only clear N (bit31) and Z (bit30) before merge */
    ld_r10d_apsr(cg);
    and_r10d_imm(cg, 0x3FFFFFFFu);     /* clear bits 30,31 */
    or_r10d_esi(cg);
    st_r10d_apsr(cg);
}
```

CRITICAL: emit_flags_nzcv must be called BEFORE st_eax. Reason: x86 EFLAGS are clobbered by
ANY subsequent instruction; st_eax (mov [r15+...], eax) doesn't touch arithmetic flags but
the helper functions invoked inside emit_flags_nzcv (lahf, seto, mov-rmod-rm) do. The
sequence is: ALU op (sets EFLAGS) -> lahf+seto (samples them into AH+CL) -> our scratch
arithmetic (clobbers EFLAGS but we already sampled) -> st_eax (no EFLAGS impact).

T32 (`OP_T32_ADD_IMM`, `OP_T32_SUB_IMM`, `OP_T32_ADDW`, `OP_T32_SUBW`): conditional on
i->set_flags. ADDW/SUBW NEVER set flags per ARM ARM (T4 encoding always uses 12-bit imm
without S bit). T32 ADD/SUB with S bit: emit flags only when set_flags.

```c
case OP_T32_ADD_IMM:
    ld_eax(cg, i->rn); add_imm(cg, i->imm);
    if (i->set_flags) emit_flags_nzcv(cg, false);
    st_eax(cg, i->rd); break;

case OP_T32_SUB_IMM:
    ld_eax(cg, i->rn); sub_imm(cg, i->imm);
    if (i->set_flags) emit_flags_nzcv(cg, true);
    st_eax(cg, i->rd); break;

case OP_T32_ADDW: case OP_T32_SUBW:
    /* T4 encoding: 12-bit imm, no S bit, no flag update */
    ld_eax(cg, i->rn);
    if (i->op == OP_T32_ADDW) add_imm(cg, i->imm); else sub_imm(cg, i->imm);
    st_eax(cg, i->rd); break;
```

- Z: result zero (a=5, b=5, op=SUB -> 0, Z=1)
- C add: carry-out (a=0xFFFFFFFF, b=1, op=ADD -> 0, C=1, V=0, Z=1)
- C sub: no-borrow (a=10, b=5, op=SUB -> 5, C=1)
- C sub: borrow (a=5, b=10, op=SUB -> 0xFFFFFFFB, C=0)
- V add: signed overflow (a=0x7FFFFFFF, b=1, op=ADD -> 0x80000000, V=1, N=1)

```c
#include "test_harness.h"
#include "core/cpu.h"
#include "core/bus.h"
#include "core/codegen.h"
#include "core/decoder.h"
#include "core/jit.h"
#include <string.h>

static jit_t s_jit;

extern bool execute(cpu_t* c, bus_t* bus, const insn_t* i);

static void run_pair(opcode_t op, u32 a, u32 b, u8 set_flags,
                     bool reg_form, u32 imm, const char* tag) {
    bus_t bus; bus_init(&bus);
    bus_add_flat(&bus, "flash", 0x00000000u, 4096u, false);
    bus_add_flat(&bus, "sram",  0x20000000u, 4096u, true);

    cpu_t c1 = {0}, c2 = {0};
    cpu_reset(&c1, CORE_M4); cpu_reset(&c2, CORE_M4);
    c1.r[1] = a; c1.r[2] = b; c1.r[REG_PC] = 0u; c1.apsr = 0u;
    c2.r[1] = a; c2.r[2] = b; c2.r[REG_PC] = 0u; c2.apsr = 0u;

    insn_t ins = {0};
    ins.op = op; ins.rd = 0u; ins.rn = 1u; ins.rm = 2u;
    ins.imm = imm; ins.set_flags = set_flags ? 1 : 0;
    ins.pc = 0u; ins.size = (op >= OP_T32_BL) ? 4u : 2u;

    /* native */
    memset(&s_jit, 0, sizeof s_jit);
    jit_init(&s_jit);
    cg_thunk_t fn = codegen_emit(&s_jit.cg, &ins, 1u);
    if (fn) (void)fn(&c1, &bus);

    /* interp */
    (void)execute(&c2, &bus, &ins);

    if (c1.apsr != c2.apsr || c1.r[0] != c2.r[0]) {
        fprintf(stderr, "MISMATCH %s op=%d a=%08x b=%08x: "
                        "native r0=%08x apsr=%08x  interp r0=%08x apsr=%08x\n",
                tag, (int)op, a, b, c1.r[0], c1.apsr, c2.r[0], c2.apsr);
    }
    TEST(c1.apsr == c2.apsr, tag);
    TEST(c1.r[0] == c2.r[0], tag);
}

int main(void) {
    /* T1 ADD/SUB always set flags: */
    run_pair(OP_ADD_REG, 0x7FFFFFFFu, 1u,        1, true, 0u, "ADD_REG +1 overflow");
    run_pair(OP_ADD_REG, 0xFFFFFFFFu, 1u,        1, true, 0u, "ADD_REG carry-out");
    run_pair(OP_ADD_REG, 5u, 5u,                 1, true, 0u, "ADD_REG no-carry");
    run_pair(OP_ADD_REG, 0u, 0u,                 1, true, 0u, "ADD_REG zero");
    run_pair(OP_SUB_REG, 0u, 1u,                 1, true, 0u, "SUB_REG borrow N");
    run_pair(OP_SUB_REG, 5u, 5u,                 1, true, 0u, "SUB_REG zero C-set");
    run_pair(OP_SUB_REG, 10u, 5u,                1, true, 0u, "SUB_REG no-borrow");
    run_pair(OP_SUB_REG, 0x80000000u, 1u,        1, true, 0u, "SUB_REG V signed");
    run_pair(OP_AND_REG, 0xFF00FF00u, 0x00FF00FFu, 1, true, 0u, "AND_REG zero");
    run_pair(OP_AND_REG, 0xFF00FF00u, 0xF000F000u, 1, true, 0u, "AND_REG nonzero");
    run_pair(OP_ORR_REG, 0u, 0u,                 1, true, 0u, "ORR_REG zero");
    run_pair(OP_ORR_REG, 0x80000000u, 0u,        1, true, 0u, "ORR_REG N");
    run_pair(OP_EOR_REG, 0xAAAAAAAAu, 0xAAAAAAAAu, 1, true, 0u, "EOR_REG zero");

    /* T1 ADD_IMM3 always sets flags: */
    run_pair(OP_ADD_IMM3, 0x7FFFFFFFu, 0u,       1, false, 1u, "ADD_IMM3 V");
    run_pair(OP_SUB_IMM3, 5u, 0u,                1, false, 5u, "SUB_IMM3 zero");
    run_pair(OP_SUB_IMM8, 10u, 0u,               1, false, 10u, "SUB_IMM8 zero");

    /* CMP family: */
    run_pair(OP_CMP_IMM, 5u, 0u,                 0, false, 5u, "CMP_IMM zero");
    run_pair(OP_CMP_IMM, 5u, 0u,                 0, false, 7u, "CMP_IMM negative");
    run_pair(OP_CMP_REG, 5u, 5u,                 0, true, 0u, "CMP_REG zero");
    run_pair(OP_CMP_REG, 5u, 10u,                0, true, 0u, "CMP_REG borrow");
    run_pair(OP_CMP_REG, 0x80000000u, 1u,        0, true, 0u, "CMP_REG V");
    run_pair(OP_CMN_REG, 0x7FFFFFFFu, 1u,        0, true, 0u, "CMN_REG V");
    run_pair(OP_CMN_REG, 0xFFFFFFFFu, 1u,        0, true, 0u, "CMN_REG carry");
    run_pair(OP_TST_REG, 0xF0F0F0F0u, 0x0F0F0F0Fu, 0, true, 0u, "TST_REG zero");
    run_pair(OP_TST_REG, 0x80000000u, 0x80000000u, 0, true, 0u, "TST_REG N");

    /* T32 CMP/CMN: */
    run_pair(OP_T32_CMP_IMM, 0xFFFFFFFFu, 0u,    0, false, 0xFFFFFFFFu, "T32 CMP_IMM zero");
    run_pair(OP_T32_CMP_REG, 100u, 50u,          0, true, 0u, "T32 CMP_REG no-borrow");

    /* T32 AND_IMM with set_flags: */
    run_pair(OP_T32_AND_IMM, 0xFF00FF00u, 0u,    1, false, 0x00FF00FFu, "T32 AND_IMM zero");
    run_pair(OP_T32_AND_IMM, 0x80000000u, 0u,    1, false, 0x80000000u, "T32 AND_IMM N");

    /* T32 ADD_IMM with set_flags=0 should NOT update flags */
    run_pair(OP_T32_ADD_IMM, 5u, 0u,             0, false, 3u, "T32 ADD_IMM no-flag");

    PASS();
}
```
