<objective>
Add native codegen for memory ops: LDR/STR (word, byte, halfword) immediate-offset and
register-offset variants, plus LDR_LIT (PC-relative literal pool) and LDRD/STRD double-word.
Each emit calls bus_read/bus_write through a WIN64 indirect call, reloading the result from a
stack scratch slot. This unlocks FreeRTOS context switch on the native path, which is dominated
by `LDR/STR Rd, [Rn, #imm]` patterns.

Purpose: JIT-02 success criterion. Without native LDR/STR, any block containing a memory op
falls back to interpreter — and FreeRTOS hot loops are full of memory ops. Today native covers
only ~17 opcode families (MOV/ADD/SUB/AND/OR/EOR variants); after this plan, native covers
~28-30 families. Combined with 14-03/14-04 (flags + B.cond) and 14-05 (chaining), the IPS
target of 100M+ becomes reachable.

Output: src/core/codegen.c grows ~250 lines of WIN64 helper-call emission. Test proves a
round-trip STR/LDR through SRAM works through the native path with zero interpreter fallback.
</objective>

Recall from 14-01: prologue saves rcx -> r15 (cpu), rdx -> r14 (bus), and reserves 32B
shadow space at [rsp..rsp+32]. WIN64 helpers receive args in rcx/rdx/r8/r9 and require
that 32B shadow region. We REUSE the shadow space as a scratch u32 slot at [rsp+0] — the
helper writes its 32B clobber into [rsp..rsp+32], and we use the LOWER 4 bytes ([rsp+0])
as our out parameter pointer target.

Wait — that conflicts. Shadow space is FOR the helper to spill rcx/rdx/r8/r9 into. We can't
share it with our out-parameter slot. Solution: the prologue in 14-01 reserves exactly 32B.
Add a separate 8B scratch slot now via a small adjustment AT THE LDR/STR site, OR (cleaner)
bump the prologue reservation in 14-01 to 48B and reserve [rsp+32..rsp+40] as the out slot,
keeping [rsp+0..rsp+32] as shadow.

Decision: do the bump locally without touching 14-01. At each LDR site emit:
`sub rsp, 16` before the call (keeps 16B alignment: 16 is multiple of 16). Use [rsp+0] as
the out slot. After the call, `add rsp, 16`. Net cost per LDR: 8 bytes for the two adjustments.

Actually even cleaner: helpers only NEED 32B shadow if they call further into other helpers
or if they actually use it (most leaf helpers ignore it). bus_read is a leaf-ish call;
the real shadow space is for it to spill rcx/rdx/r8/r9 if it wants. We can give it both:

Final design — at each LDR/STR site, the WIN64 sequence is:

```asm
; LDR Rd, [Rn, #imm]
; (assume r15=cpu, r14=bus, rsp 16B-aligned with 32B shadow already reserved by prologue)

mov  eax, [r15 + R_OFF + rn*4]   ; addr base
add  eax, imm                     ; addr = base + imm
sub  rsp, 16                      ; 16B-aligned local: out_slot at [rsp+0], rest unused
mov  rcx, r14                     ; arg0 = bus
mov  edx, eax                     ; arg1 = addr (zero-extend implicit on 32->64)
mov  r8d, 4                       ; arg2 = size
lea  r9, [rsp + 0]                ; arg3 = &out_slot
mov  rax, &bus_read               ; absolute imm64 -> rax
call rax                          ; bus_read(bus, addr, 4, &out)
; bool in al; out u32 at [rsp+0]
test al, al
jz   <halt_path>
mov  eax, [rsp + 0]               ; load fetched word
add  rsp, 16                      ; restore rsp
mov  [r15 + R_OFF + rd*4], eax   ; cpu->r[rd] = result
```

For LDRB: `mov r8d, 1` and after load mask `and eax, 0xFF` (zero-extend semantics — ARM LDRB
zero-extends; LDRSB sign-extends, NOT in this plan).
For LDRH: `mov r8d, 2` and `and eax, 0xFFFF`.

For STR:

```asm
mov  eax, [r15 + R_OFF + rn*4]   ; base
add  eax, imm                     ; addr
sub  rsp, 16
mov  rcx, r14                     ; bus
mov  edx, eax                     ; addr
mov  r8d, 4                       ; size
mov  r9d, [r15 + R_OFF + rd*4]   ; val (zero-extend into r9)
mov  rax, &bus_write
call rax
test al, al
jz   <halt_path>
add  rsp, 16
```

For STRB: `mov r8d, 1`. STRH: `mov r8d, 2`. The val source is the same r[rd]; bus_write only
reads the low size bytes per its impl, but to be defensive we mask r9d before the call:
`and r9d, 0xFF` for B / `and r9d, 0xFFFF` for H.

Halt path (shared per-block, emit ONCE near epilogue):
```asm
.halt:
mov  byte [r15 + CG_HALT_OFF], 1   ; cpu->halted = 1
add  rsp, 16                        ; balance the unsuccessful sub
add  rsp, 32                        ; balance prologue shadow
pop  rsi; pop rbx; pop r14; pop r15
xor  al, al                          ; return false
ret
```

Key trade: emitting one halt path per BLOCK keeps it simple. Each LDR/STR `jz` jumps to a
fixed displacement. Track the halt-path offset; back-patch jcc targets when the block ends.
For Phase 14, simpler still: emit a SHORT (2-byte rel8) `jz +N` to a per-instruction inline
halt stub. That makes each LDR ~5 bytes longer but no back-patching needed.

Cleanest approach for this plan: emit a per-thunk halt block at the END of the thunk (after
the success epilogue). Use rel32 jcc (6 bytes per branch) so we can compute the displacement
by tracking `cg->used` of the halt block at emit time. Simpler design without back-patching:

1. emit_op walks the insn array. For each LDR/STR, emit the call + `jz halt_rel32` where
   rel32 = (halt_pos - jcc_end). But halt_pos is unknown until end of block.
2. Solution A: TWO-PASS. First pass measures bytes, second emits.
3. Solution B: emit a SHORT BACKWARD branch that ALWAYS jumps if we eventually need halt.
   No — we don't yet know if any LDR will fail at runtime.
4. Solution C: emit halt stub FIRST in the block (relative jumps backward known at emit), then
   prologue, then ops. Halt stub at known offset. Each LDR/STR's `jz` knows the target.

Use Solution C — emit a tiny halt stub at the very start of the thunk, BEFORE the prologue,
with an unconditional `jmp` over it to the real prologue. Halt stub still needs prologue
state to know rsp delta, but it operates on REAL cpu pointer in rcx (still rcx at thunk
entry — no, by the time a LDR fails inside the body, prologue ran, rsp was already shifted).

Cleanest: track failures into a non-volatile register. Use rbx (saved by prologue, free for
us). Initialize rbx=0 at prologue end. Every LDR/STR sets rbx=1 if al==0 and skips the
register store via a forward `jz` to the next instruction's start. After all insns: at
epilogue, check rbx; if rbx==1, set halted and return false; else normal success.

```asm
; Modified prologue (extension to 14-01 baseline): zero rbx as failure flag
; (xor ebx, ebx is 31 DB, 2 bytes; emit at end of emit_prologue or top of emit_op for first LDR)
xor  ebx, ebx       ; 31 DB

; per-LDR failure path: skip remaining ops if al==0
; after call rax:
test al, al         ; 84 C0
jz   .ldr_fail      ; 0F 84 rel32 (forward branch; rel32 patched after epilogue location known)
; success path follows
```

OK this is getting complex. Final pragmatic decision: use a SIMPLE 2-byte rel8 forward branch
to a fail-path located IMMEDIATELY after each LDR/STR. The fail path: `mov bl, 1; jmp <skip>`
where skip = past the success-side store. Per instruction:

```asm
; ... call rax (bus_read)
test al, al                ; 84 C0  (2B)
jnz  .ok                    ; 75 NN   (2B; skip the 6-byte fail body)
mov  bl, 1                  ; B3 01   (2B)
jmp  .end_op                ; EB MM   (2B; skip success store)
.ok:
mov  eax, [rsp + 0]         ; success: load result
add  rsp, 16
mov  [r15 + R_OFF + rd*4], eax
.end_op:
```

But then on the success path we still need to add rsp, 16. Let `add rsp, 16` come BEFORE the
test (always pop the local slot regardless of outcome):

```asm
; ... bus_read returned; al=0/1; out at [rsp+0]
mov  ecx, [rsp + 0]         ; save out value into ecx (safe: ecx is volatile and we're done with it)
add  rsp, 16                ; restore rsp unconditionally
test al, al                 ; check return
jnz  .ok                    ; 75 NN
or   bl, 1                  ; mark failure
jmp  .skip_store
.ok:
mov  [r15 + R_OFF + rd*4], ecx
.skip_store:
```

For STR: no result to store, but still need fail flag:
```asm
; ... bus_write returned; al=0/1
add  rsp, 16
test al, al
jnz  .ok
or   bl, 1
.ok:                         ; nothing to store, just continue
```

Step B — emit_ldr_imm core sequence. Centralize the LDR pattern; size 1/2/4 controlled by
caller, mask applied for B/H zero-extend.

```c
/* LDR Rd, [Rn, #imm]:  rd <- bus_read(bus, r[rn] + imm, sz), zero-extended.
   sz is 1/2/4. Emits ~46 bytes. */
static void emit_load(codegen_t* cg, u8 rd, u8 rn, u32 imm, u32 sz) {
    ld_eax(cg, rn);                        /* eax = r[rn]               (7B) */
    add_imm(cg, imm);                      /* eax += imm                (5B) */
    sub_rsp_16(cg);                        /* rsp -= 16                 (4B) */
    mov_rcx_r14(cg);                       /* rcx = bus                 (3B) */
    mov_edx_eax(cg);                       /* rdx = addr                (2B) */
    mov_r8d_imm(cg, sz);                   /* r8d = sz                  (6B) */
    lea_r9_rsp0(cg);                       /* r9  = &out                (4B) */
    mov_rax_imm64(cg, (u64)(uintptr_t)bus_read);   /* rax = &bus_read   (10B) */
    call_rax(cg);                          /* call bus_read             (2B) */
    mov_ecx_rsp0(cg);                      /* ecx = out                 (3B) */
    add_rsp_16(cg);                        /* rsp += 16                 (4B) */
    /* zero-extend mask for B/H (LDR word skips this) */
    if (sz == 1u) {
        emit_b(cg, 0x81); emit_b(cg, 0xE1); emit_w32(cg, 0xFFu);          /* and ecx, 0xFF (6B) */
    } else if (sz == 2u) {
        emit_b(cg, 0x81); emit_b(cg, 0xE1); emit_w32(cg, 0xFFFFu);        /* and ecx, 0xFFFF */
    }
    /* test al; jnz +5 (skip 'or bl,1; jmp +N') */
    test_al_jnz(cg, 5u);                   /* test;jnz +5               (4B) */
    or_bl_1(cg);                           /* or bl,1                   (3B) */
    jmp_short(cg, 4u);                     /* jmp +4 (skip the store)   (2B) */
    /* success: store ecx into r[rd]   (7B); jnz +5 lands here */
    st_ecx(cg, rd);                        /* mov [r15+...], ecx        (7B) */
    /* total ~67B per LDR */
}
```

Critical encoding detail: the `jnz +5` skips exactly 5 bytes which is `or bl,1` (3) + `jmp +4` (2) = 5. The
`jmp +4` skips 4 bytes which is the next 4 bytes of `st_ecx` — wait, st_ecx is 7 bytes, not 4.

Re-check: we want the failure path to NOT execute st_ecx. Failure path emits: `or bl,1` (3B)
+ `jmp <past-st_ecx>` which must skip ALL 7 bytes of st_ecx. So `jmp +7`. And `jnz` skips the
failure stub entirely: `or bl,1` (3) + `jmp NN` (2) = 5 bytes. So: `jnz +5`, then on failure
`or bl,1; jmp +7`, then `st_ecx` (7B). After jnz +5 we land at st_ecx. After jmp +7 we land
PAST st_ecx. Correct.

Update:
```c

    or_bl_1(cg);             /* failure: bl |= 1 */
    jmp_short(cg, 7u);       /* jump past the 7-byte st_ecx */
    st_ecx(cg, rd);          /* success path */
```

Step C — variants. Each calls emit_load/emit_store with the right size and base register:

- OP_LDR_IMM    (T1 ARMv6-M):  base = r[rn], imm = i->imm, sz=4
- OP_STR_IMM:                  base = r[rn], imm = i->imm, sz=4
- OP_LDRB_IMM:  sz=1; OP_LDRH_IMM: sz=2; OP_STRB_IMM: sz=1; OP_STRH_IMM: sz=2
- OP_LDR_SP / OP_STR_SP:  base = r[REG_SP] (rn=13), imm = i->imm, sz=4
- OP_LDR_LIT:   addr = ((i->pc + 4) & ~3u) + i->imm; emit as `mov eax, addr_const` directly
                (no rn load); then proceed with the bus_read sequence.
- OP_LDR_REG:   addr = r[rn] + r[rm]; emit `ld_eax(rn); ld_ecx(rm); add eax,ecx;` then call
- OP_STR_REG:   same but with bus_write
- OP_T32_LDR_IMM, OP_T32_STR_IMM, OP_T32_LDRB_IMM, etc.: T32 wide forms with i->imm 12-bit.
                Encoding signal in i->imm directly; same emit path.

                   i->rd, i->rd2 (check decoder.h for the second-reg field name).
                   emit_load(rd, rn, imm, 4); emit_load(rd2, rn, imm+4, 4);
                   But the second emit recomputes r[rn]+imm+4 from scratch — fine, simple.
- OP_T32_STRD_IMM: same with two emit_store.

    for (u8 k = 0; k < n; ++k) if (!codegen_supports(ins[k].op)) return NULL;
    if (cg->used + (u32)n * 96u + 64u > cg->capacity) return NULL;
    u8* start = cg->buffer + cg->used;
